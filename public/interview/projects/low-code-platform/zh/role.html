02我的职责

我负责什么，不负责什么

我主导的部分

  • 项目整体技术架构——编辑器、渲染引擎、组件库、构建发布链路四条线的方案与落地。
  • JSON schema 的设计本身，以及渲染引擎的契约（一份 schema 如何变成组件树 + 触发器图）。
  • 组件库的对外 API、版本管理机制、CDN 加载策略（每个 widget 独立 bundle，带 semver）。
  • 平台推广落地——直接对接 4 条业务线的产品与运营，收集反馈、驱动切换。
  • 团队的工程规范、脚手架、内部文档和代码评审标准。

我协作但不主导的部分

  • Schema 存储、版本管理、发布工作流的后端服务——由后端同学实现，我负责定义 API 契约。
  • 视觉和品牌语言——由设计同学定义，我负责让组件库忠实实现，并在 schema 层把关不允许「破坏性配置」。
  • 实验与埋点 SDK——我设计了触发器的 hook 点，平台组负责接入和数据回流。

团队：我作为技术负责人 + 3–5 位前端，根据阶段弹性调整。之所以我要同时管编辑器、运行时和推广三件事—— 因为这类平台的关键设计决策，单看任何一面都会走偏，只有三面一起看才成立。
