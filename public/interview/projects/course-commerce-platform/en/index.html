01Overview

What it is, in one sentence

A low-code admin system with a visual page builder inside it, so non-engineers on the ops team can ship production course landing pages end-to-end — from assembling the page to configuring forms, previewing on a phone, and hitting publish.

Situation
Every new course needed a new promo landing page. About 80% of the page structure was the same every time — hero, course intro, teacher, pricing, sign-up form. The only real difference was the copy and the course. But because it was all hand-coded, every page took the frontend team 3–5 days, and ops had to queue up for us.
Task
Move the "building a page" capability from engineering over to ops, without losing page quality, load speed, or design consistency. Basically: let them self-serve, but keep the output production-grade.
Action
Built a 7-step wizard that walks ops through basic info → materials → course binding → rules → form → review → share. Plugged in Formily so any form field on the page is just a config, not code. Added an iPhone-frame preview on the right side so ops can see exactly what the user will see, on the correct screen size, with a QR code they can scan to test on a real phone.
Result
Ops ships their own course pages in a few hours instead of waiting several days. The frontend team stopped being the bottleneck and shifted focus from "making pages" to "making the tool better." The platform also grew beyond just the builder — template center, component library, product/coupon center, operation tools, and user segmentation all got pulled into the same admin.
