03My Role & Scope

What I owned, and what I didn't

I owned

  • Frontend architecture of the admin tool — built on Umi Max + React + TypeScript + Ant Design Pro.
  • The page builder itself: the 7-step wizard flow, drag-and-drop component sorting, the Formily-driven form configuration, and the save / preview / publish pipeline.
  • The iPhone preview panel — multi-device frames (regular / short / long), live re-rendering on config change, and QR code so ops can scan and test on a real phone.
  • Shared component library used across the admin: sortable list, upload widgets, Formily form wrappers, and the device simulator.
  • Multi-environment build setup (dev / grayscale / production) and the release workflow.

I partnered on / didn't own

  • Backend APIs for page storage, template persistence, and CDN publishing — owned by the backend team. I defined the client contracts.
  • The underlying design system — based on Ant Design, with a few internal tweaks from the design team.
  • Course, coupon, and product data models — owned by product and backend. I consumed them.
  • Permissions — based on Umi Max's access plugin, configured together with backend.

Team: small squad working closely with ops, PM, design, and the backend team. My lane was the whole frontend surface of the admin; the data layer and publishing pipeline were a shared effort.
